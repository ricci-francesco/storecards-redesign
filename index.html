StoreCard Anatomy

Modules, Elements & Context Dimensions

A StoreCard is a UI component made of independent modules. Each module contains elements rendered in a configurable priority order. The right config is selected based on two context dimensions.

Dimension 1
Surface
Which screen of the app the StoreCard is rendered on.
StoreWall Search DiscoveryWall
Dimension 2
Vertical
The business category of the stores shown in that surface.
Food Groceries Retail
Config unit
Module
A group of semantically related UI elements inside a StoreCard. Each module has a dedicated space and an internal priority hierarchy.
Config unit
Element
A single piece of information inside a module. The highest-priority available element in the hierarchy is the one rendered.
Highlights
One tag showing store's key differentiator. Hierarchy: Exclusivity → TopRated → New
Trust Markers
Social or quality signal. Hierarchy: FriendsOrders → MostPopular → InStorePricing
Core Info
Always-on module. StoreName · Rating · ETA · DeliveryFee · Sponsored
Promotions
Active discount or offer. Position is configurable (overlay or dedicated row).
